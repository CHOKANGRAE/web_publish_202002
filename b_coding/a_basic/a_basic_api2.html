div는 의미가 없는 단순한 영역을 나타내게 하는 박스의 의미

span 요소는 의미가 없는 단순한 내용의 그룹 (div와는 조금 다른 의미)

시멘틱 구조는, 각 영역별로 그 의미를 가지고 쓰이도록 처리된 영역

header, nav, section, article, aside, main, footer, figure

영역을 나타내는 의미를 가진 구조

시멘틱 구조 이해하기

header

header : 웹 페이지 상단에 존재, 웹의 로고/h1을 담으면서, 전체 웹페이지 구조를 파악할 수 있는 nav 영역도 함께 포함시키는 기능

navigation

nav : 웹 페이지 내부에서 클릭하면 해당하는 페이지로 이동하게 만들어주는 링크들의 집합 / navigation 요소를 담아놓은 영역


main : section, article의 영역(header, footer 제외)을 정리해서 더 큰 영역으로 담는 곳
아직 대부분의 사이트에서는 완벽하게 지원하지 못하고 있다.

하나의 대표 내용을 보여주는 영역

section : 하나의 대표 내용을 보여주는 영역으로, 다른 내용과 연계가 되어있는 부분, 본문 / 광고(배너)

다른 내용과 연계가 되지 않아도 별도의 독립영역

article : section과 유사한 기능을 가진 영역, 본문내용, 다른 내용과 연계가 되지 않아도 별도의 독립영역으로 설정되어도 문제 없는 영역 / 게시판 안내, 신문기사, 광고

another 영역을 지칭

aside : another영역을 지칭, 사이트의 구성요소에 없어도 전혀 문제되지 않는 영역, 있으면 도움이 되는 영역 /


사이트 정보

footer : 이름으로 해석하면 하단에 존재한다는 의미를 가지고 있지만, 실제로는 하단에만 존재하지 않음
footer는 회사/사이트의 정보를 담고 있는 영역을 지칭 / 주소, 연락처, 제작연도, 지침, family site
정보가 보이지 않는다고 하더라도, 사이트에 대한 내용은 반드시 존재해야함
